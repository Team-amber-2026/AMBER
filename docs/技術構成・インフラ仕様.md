# 技術構成・インフラ仕様

## 構成
- **Frontend**: React（Vite） -> Vercel
- **Backend**: Python（Django） -> Render Web Service
- **Database**: PostgreSQL（Render Managed DB）
- **Storage**: Cloudinaryを第一候補とする。レシート画像保存用に利用する。

## 環境変数（.env）管理
- `SECRET_KEY`: Djangoのシークレットキー
- `DEBUG`: 開発環境ではTrue、本番環境ではFalse
- `DATABASE_URL`: DB接続情報（Render Web Service と Render PostgreSQL が同一workspace・同一regionの場合は internal connection string）
- `GOOGLE_APPLICATION_CREDENTIALS`: OCR認証情報
- `CORS_ALLOWED_ORIGINS`: フロントエンドのURLを許可
- `CSRF_TRUSTED_ORIGINS`: CSRFを許可するフロントエンドURL
- `CLOUDINARY_URL`: Cloudinary接続情報

## 注意点
- `.env` はGitHubにアップロードしない。
- Google Cloud Vision APIの認証情報はバックエンド側で管理する。
- React側にAPIキーを置かない。
- 本番環境では `DEBUG=False` にする。
- Vercel側には `VITE_API_BASE_URL=https://<Render Web Service URL>/api` を設定する。
- Render側には Vercel のURLを `FRONTEND_ORIGIN` / `CORS_ALLOWED_ORIGINS` / `CSRF_TRUSTED_ORIGINS` に設定する。
- Render PostgreSQL の internal connection string は、同一workspace・同一regionのRenderサービスからのみ利用する。

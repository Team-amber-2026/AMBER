# 技術構成・インフラ仕様

## 構成
- **Frontend**: React（Vite） -> Render Static Site
- **Backend**: Python（Django） -> Render Web Service
- **Database**: PostgreSQL（Render Managed DB）
- **Storage**: Cloudinaryを第一候補とする。レシート画像保存用に利用する。

## 環境変数（.env）管理
- `SECRET_KEY`: Djangoのシークレットキー
- `DEBUG`: 開発環境ではTrue、本番環境ではFalse
- `DATABASE_URL`: DB接続情報
- `GOOGLE_APPLICATION_CREDENTIALS`: OCR認証情報
- `CORS_ALLOWED_ORIGINS`: フロントエンドのURLを許可
- `CSRF_TRUSTED_ORIGINS`: CSRFを許可するフロントエンドURL
- `CLOUDINARY_URL`: Cloudinary接続情報

## 注意点
- `.env` はGitHubにアップロードしない。
- Google Cloud Vision APIの認証情報はバックエンド側で管理する。
- React側にAPIキーを置かない。
- 本番環境では `DEBUG=False` にする。

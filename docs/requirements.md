## 2. プロダクト概要

**ためるん** は、レシートを撮影またはアップロードするだけで、支出を記録し、今月の支出合計を確認できる家計簿Webアプリである。

ユーザーはスマホブラウザからレシート画像を登録し、OCRによって読み取られた店名・購入日・合計金額・カテゴリーを確認、必要に応じて修正した上で保存する。保存された支出データは一覧画面や月次集計画面で確認できる。

---

## 3. 解決したい課題

家計簿をつけたいと思っていても、レシートの内容を毎回手入力するのが面倒で、支出の記録が続かない。

---

## 4. 想定ユーザー

| ユーザー | 内容 |
|---|---|
| メインユーザー | 家計管理をしたい人 |
| 利用シーン | 買い物後にレシートを撮影し、支出を記録する |
| 困っていること | 手入力が面倒、入力を忘れる、今月の支出が分かりにくい |
| 期待すること | レシート画像から簡単に支出を記録し、月ごとの支出を確認できること |

---

## 5. プロダクトの目的

### 5.1 目的

レシート入力の手間を減らし、家計簿の記録を継続しやすくする。

### 5.2 提供価値

- レシート撮影・アップロードだけで支出登録の入力負担を減らす
- OCR結果を確認・修正できるため、読み取りミスがあっても使える
- 今月の支出合計をすぐに確認できる
- 紙のレシートを見返さなくても支出履歴を確認できる

---

## 6. MVP定義

MVPでは、以下の一連の流れが実装されている状態を完成とみなす。

```text
ログイン
 ↓
レシート画像を撮影またはアップロード
 ↓
OCRで文字を読み取る
 ↓
日付・店名・合計金額・カテゴリーを確認する
 ↓
必要があれば修正する
 ↓
保存する
 ↓
今月の支出合計に反映される
```

### 6.1 MVPで必ず実装する範囲

| 分類 | 内容 |
|---|---|
| 認証 | ユーザー登録、ログイン、ログアウト |
| レシート登録 | スマホから画像撮影または画像選択 |
| OCR | Google Cloud Vision APIによる文字読み取り |
| 確認・修正 | OCR結果の確認、手動修正 |
| 支出管理 | 支出データの保存、一覧、詳細、編集、削除 |
| 集計 | 月次合計、カテゴリー別合計 |
| スマホ対応 | スマホブラウザで利用しやすいレスポンシブUI |

### 6.2 MVPでは実装しない範囲

| 機能 | 理由 |
|---|---|
| 銀行・カード連携 | セキュリティや外部サービス連携の難易度が高いため |
| 予算設定機能 | MVPの範囲を超えるため |
| 家族共有機能 | 権限管理が複雑になるため |
| 商品ごとの細かい分類 | OCR処理と分類処理の難易度が高くなるため |
| AIによる節約アドバイス | メイン機能ではないため |
| ネイティブスマホアプリ | 開発難易度が上がるため、今回はスマホ対応Webアプリとする |

---

## 7. システム利用イメージ

1. ユーザーがスマホまたはPCからWebアプリにアクセスする
2. ユーザー登録またはログインを行う
3. レシート登録画面でレシート画像を撮影またはアップロードする
4. OCR解析を実行する
5. 解析された店名・購入日・合計金額・カテゴリーを確認する
6. 誤りがある場合は手動で修正する
7. 保存ボタンを押して支出データとして登録する
8. ホーム画面や月次集計画面で支出合計を確認する

## 16. スマホ対応要件

本アプリはネイティブスマホアプリではなく、スマホ対応Webアプリとして実装する。

### 16.1 UI要件

- スマホ幅では1カラムレイアウトにする
- ボタンや入力欄はタップしやすい大きさにする
- 支出一覧はカード形式で表示する
- 横スクロールが発生しないようにする
- 月次集計は縦並びのカードまたはバー表示にする

### 16.2 カメラ・画像要件

- `input type="file"` を利用して画像選択に対応する
- `accept="image/*"` を指定する
- スマホでは `capture="environment"` を利用し、外向きカメラでの撮影を想定する
- カメラが起動しない場合に備えて、画像アップロードにも対応する
- 画像選択後はプレビューを表示する

### 16.3 動作確認要件

- スマホブラウザでログインできること
- スマホから画像を選択または撮影できること
- OCR解析を実行できること
- OCR結果を修正できること
- 支出データを保存できること
- 月次合計に反映されること

## 19. テスト観点

### 19.1 認証

- スマホで画像を選択できる
- カメラ撮影ができる
- OCR解析結果が表示される
- OCR失敗時にエラーが表示される
- OCR結果を修正できる

### 19.3 支出管理

- 支出を保存できる
- 支出一覧に表示される
- 支出詳細を確認できる
- 支出を編集できる
- 支出を削除できる

### 19.4 集計

- 今月の合計金額が表示される
- カテゴリー別合計が表示される

## 20. 受け入れ条件

卒業制作のMVPとして、以下を満たした場合に完成とみなす。

- ユーザー登録、ログイン、ログアウトができる
- ログイン中のユーザーのみ支出データを扱える
- スマホからレシート画像を登録できる
- OCRで文字を読み取れる
- OCR結果を確認・修正できる
- 確認済みデータを支出として保存できる
- 支出一覧を表示できる
- 今月の支出合計を表示できる
- カテゴリー別集計を表示できる
- Vercel / Render上にデプロイし、スマホ実機からアクセスできる

- APIキー、パスワード、認証情報はGitHubへアップロードしない

## 23. 発表デモ要件

発表では以下の流れを見せる。

1. ユーザーがログインする
2. レシート画像を撮影またはアップロードする
3. OCRによって店名・日付・合計金額が表示される
4. ユーザーが内容を確認・修正する
5. 支出データとして保存する
6. ホーム画面または月次集計画面で合計金額に反映されることを確認する

発表では、個人情報を含まないサンプルレシート画像を使用する。
